## Hetzner CLI Commands

Full Hetzner lifecycle from your terminal. `ankra cluster hetzner create` provisions a cluster with configurable network ranges, server types, and k3s versions. Scale workers with `ankra cluster hetzner scale`, upgrade Kubernetes with `ankra cluster hetzner upgrade`, and tear everything down with `ankra cluster hetzner deprovision`. Manage Hetzner API tokens and SSH keys via `ankra credentials hetzner` — including automatic keypair generation.

```bash
# Create credentials
ankra credentials hetzner create --name my-hetzner-token
ankra credentials hetzner ssh-key create --name my-ssh-key --generate

# Create a cluster
ankra cluster hetzner create \
  --name my-cluster \
  --credential-id <hetzner-credential-id> \
  --ssh-key-credential-id <ssh-key-credential-id> \
  --location fsn1 \
  --control-plane-count 1 \
  --control-plane-server-type cx33 \
  --worker-count 2 \
  --worker-server-type cx33

# Scale workers
ankra cluster hetzner scale <cluster_id> 4

# Upgrade Kubernetes
ankra cluster hetzner upgrade <cluster_id> v1.35.1+k3s1

# Deprovision
ankra cluster hetzner deprovision <cluster_id>
```

## Hetzner REST API

Every Hetzner operation is available via the REST API with Bearer token authentication. Create and deprovision clusters, scale workers, upgrade k3s, and manage Hetzner credentials and SSH keys programmatically. Integrate Hetzner cluster provisioning into your CI/CD pipelines or custom tooling.

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/v1/clusters/hetzner` | POST | Create a Hetzner cluster |
| `/api/v1/clusters/hetzner/{id}` | DELETE | Deprovision a cluster |
| `/api/v1/clusters/hetzner/{id}/worker-count` | GET | Get worker count |
| `/api/v1/clusters/hetzner/{id}/scale-workers` | POST | Scale workers |
| `/api/v1/clusters/hetzner/{id}/k8s-version` | GET | Get Kubernetes version |
| `/api/v1/clusters/hetzner/{id}/upgrade-k8s-version` | POST | Upgrade Kubernetes version |
| `/api/v1/credentials/hetzner` | GET/POST | List/create Hetzner credentials |
| `/api/v1/credentials/hetzner/ssh-keys` | GET | List SSH key credentials |
| `/api/v1/credentials/hetzner/ssh-key` | POST | Create SSH key credential |

## AI Incident Reports & Root Cause Analysis

When an alert triggers, Ankra automatically generates a detailed AI incident report — no manual investigation required. The report includes root cause analysis, recommended actions, and direct links to continue troubleshooting in the AI chat.

**Automated Analysis Pipeline**:
1. Alert triggers and the system collects cluster data (pod status, Kubernetes events, container logs, job results)
2. AI performs deep analysis to identify the root cause, quoting exact errors from logs and events
3. A structured report is generated with severity assessment, affected resources, and actionable recommendations
4. The report is stored and accessible from the Alerts dashboard, cluster overview, and notification channels

**Analysis Report Contents**:
- **Root Cause**: 2-3 sentence diagnosis with evidence from logs and events
- **Severity**: Critical, warning, or info classification
- **Affected Resources**: List of impacted pods, deployments, and related resources
- **Recommended Actions**: 3-5 specific, actionable steps to resolve the issue
- **Conversation Starters**: Pre-built questions to continue investigating in the AI chat
- **Confidence Score**: AI self-assessment of analysis accuracy (0.1-1.0)

**Notification Integration**: Incident reports are delivered alongside alert notifications in Slack, Teams, PagerDuty, Discord, and Opsgenie.

## SOPS Integration with ArgoCD

End-to-end secret encryption using SOPS with AGE keys, fully integrated with ArgoCD for automatic decryption during GitOps sync. Encrypt specific YAML keys while keeping everything else readable.

**AGE Key Management**: Generate AGE key pairs at the organisation level. Private key stored securely in Vault. Automatically deployed to ArgoCD namespace.

**ArgoCD Integration**: Helm-secrets plugin and SOPS binary configured on the ArgoCD repo-server. Encrypted values are decrypted on-the-fly during Helm rendering.

```bash
ankra cluster encrypt manifest my-secret --key DB_PASSWORD -f cluster.yaml
ankra cluster encrypt addon --name grafana --key adminPassword -f cluster.yaml
ankra cluster decrypt manifest my-secret -f cluster.yaml
```
