# Helm, Kubernetes Resources & Security Hardening

Helm management, Kubernetes resource access, direct cluster selection, and client hardening.

## Helm Release Commands

Commands for managing Helm releases in the active cluster:
- `ankra cluster helm releases` ‚Äî List Helm releases (`--namespace`, `--all-namespaces`, `--output json`)
- `ankra cluster helm uninstall <release> --namespace <ns>` ‚Äî Uninstall a Helm release

## Helm Registry Commands

- `ankra helm registries list` ‚Äî List Helm registries
- `ankra helm registries get <name>` ‚Äî Get registry details
- `ankra helm registries create --name <name> --url <url>` ‚Äî Create a registry
- `ankra helm registries delete <name>` ‚Äî Delete a registry

## Helm Credential Commands

- `ankra helm credentials list` ‚Äî List registry credentials
- `ankra helm credentials create --name <name>` ‚Äî Create a credential with secure prompts
- `ankra helm credentials delete <name>` ‚Äî Delete a credential

## Kubernetes Resource Commands

Commands for accessing Kubernetes resources in the active cluster:
- `ankra cluster pods` ‚Äî List pods (`--namespace`, `--all-namespaces`, `--output json`)
- `ankra cluster resources <kind>` ‚Äî Get resources by kind (`--namespace`, `--name`, `--output json`)
- `ankra cluster logs <pod>` ‚Äî Stream pod logs (`--namespace`, `--container`, `--follow`, `--tail`, `--previous`)

## Cluster Selection & Info

- `ankra cluster select <name>` ‚Äî Select a cluster directly by name without the interactive picker
- `ankra cluster info [name]` ‚Äî Show cluster details (defaults to selected cluster; replaces `cluster get`)

## Security & Reliability

- HTTP client timeouts and response body size limits
- URL encoding for all API path and query parameters
- Config file permissions restricted to owner-only (0600)
- Machine ID hashed with SHA-256
- Browser URL validation before opening
</Update>

# Node Groups, UpCloud & OVH

Node group management commands and new UpCloud/OVH commands for cluster and credential management.

## Node Group Commands

Available for all three providers (`hetzner`, `ovh`, `upcloud`):
- `ankra cluster <provider> node-group list <cluster_id>` ‚Äî List node groups
- `ankra cluster <provider> node-group add <cluster_id> --name <name> --instance-type <type> --count <n>` ‚Äî Add a group
- `ankra cluster <provider> node-group scale <cluster_id> <group_name> <count>` ‚Äî Scale a group
- `ankra cluster <provider> node-group upgrade <cluster_id> <group_name> <instance_type>` ‚Äî Upgrade instance type
- `ankra cluster <provider> node-group delete <cluster_id> <group_name>` ‚Äî Delete a group

## UpCloud Commands

**Credentials**:
- `ankra credentials upcloud list` ‚Äî List UpCloud API credentials
- `ankra credentials upcloud create --name <name>` ‚Äî Create credential with secure token prompt
- `ankra credentials upcloud ssh-key list` ‚Äî List SSH key credentials
- `ankra credentials upcloud ssh-key create --name <name> --generate` ‚Äî Create SSH key

**Clusters**:
- `ankra cluster upcloud create` ‚Äî Create an UpCloud cluster
- `ankra cluster upcloud deprovision <cluster_id>` ‚Äî Deprovision a cluster
- `ankra cluster upcloud workers <cluster_id>` ‚Äî Get worker count
- `ankra cluster upcloud scale <cluster_id> <count>` ‚Äî Scale workers
- `ankra cluster upcloud k8s-version <cluster_id>` ‚Äî Get Kubernetes version
- `ankra cluster upcloud upgrade <cluster_id> <version>` ‚Äî Upgrade Kubernetes version

## OVH Commands

**Credentials**:
- `ankra credentials ovh list` ‚Äî List OVH API credentials
- `ankra credentials ovh create --name <name> --project-id <id>` ‚Äî Create credential with secure prompts
- `ankra credentials ovh ssh-key list` ‚Äî List SSH key credentials
- `ankra credentials ovh ssh-key create --name <name> --generate` ‚Äî Create SSH key

**Clusters**:
- `ankra cluster ovh create` ‚Äî Create an OVH cluster
- `ankra cluster ovh deprovision <cluster_id>` ‚Äî Deprovision a cluster
- `ankra cluster ovh workers <cluster_id>` ‚Äî Get worker count
- `ankra cluster ovh scale <cluster_id> <count>` ‚Äî Scale workers
- `ankra cluster ovh k8s-version <cluster_id>` ‚Äî Get Kubernetes version
- `ankra cluster ovh upgrade <cluster_id> <version>` ‚Äî Upgrade Kubernetes version

# SOPS Encryption & API Tokens

New SOPS encryption commands and API token management.

## SOPS Encryption

- `ankra cluster sops-config` ‚Äî Display the SOPS encryption configuration
- `ankra cluster encrypt manifest <name> --key <key> -f <file>` ‚Äî Encrypt a manifest key
- `ankra cluster encrypt addon --name <addon> --key <key> -f <file>` ‚Äî Encrypt an addon key
- `ankra cluster decrypt manifest <name> -f <file>` ‚Äî Decrypt a manifest

## API Tokens

- `ankra tokens list` ‚Äî List API tokens
- `ankra tokens create <name>` ‚Äî Create a token
- `ankra tokens revoke <id>` ‚Äî Revoke a token
- `ankra tokens delete <id>` ‚Äî Delete a revoked token
</Update>
