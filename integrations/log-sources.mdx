---
title: Log Sources
description: Connect a log store or an external log viewer so you can jump from any pod straight to its logs - and let Ankra AI read them.
---

Ankra ships a built-in [log viewer](/essentials/logs) that streams pod logs live. On top of that, the cluster **Integrations** hub adds two ways to wire in your existing logging stack:

- **External Log Viewer** - deep-link from any pod to its logs in Grafana, Kibana, Cloud Logging, or any URL-addressable viewer.
- **Log Source** - connect a queryable log store (Loki, Elasticsearch, OpenSearch) so Ankra AI can read logs directly during troubleshooting.

Both are configured under a cluster's **Settings → Integrations**. See [Cluster Integrations](/essentials/cluster-settings#integrations) for the hub overview.

---

## External Log Viewer

Point Ankra at the log tool you already use and it renders a **View logs** link on every pod that resolves the pod, namespace, and cluster into your viewer's URL.

### URL template

Provide a URL template with any of these placeholders, which Ankra substitutes per pod:

| Placeholder | Resolves to |
|-------------|-------------|
| `{{pod_name}}` | The pod's name |
| `{{namespace}}` | The pod's namespace |
| `{{cluster_name}}` | The cluster's name |

<CodeGroup>

```text Grafana Explore (Loki)
https://grafana.example.com/explore?left=%7B%22queries%22:%5B%7B%22expr%22:%22%7Bpod%3D%5C%22{{pod_name}}%5C%22,namespace%3D%5C%22{{namespace}}%5C%22%7D%22%7D%5D%7D
```

```text Kibana
https://kibana.example.com/app/discover#/?_a=(query:(query_string:(query:'kubernetes.pod.name:{{pod_name}} and kubernetes.namespace:{{namespace}}')))
```

</CodeGroup>

<Steps>
  <Step title="Open the integration">
    Go to the cluster's **Settings → Integrations → External Log Viewer**.
  </Step>
  <Step title="Paste your URL template">
    Enter the template with the placeholders above and save.
  </Step>
  <Step title="Use it from a pod">
    Open any pod and choose **View logs** to jump to your external viewer with the pod already scoped.
  </Step>
</Steps>

## Log Source

<Info>
Log Source is rolling out. When it is unavailable for your deployment, the integration shows as **Coming soon**.
</Info>

A **Log Source** connects a queryable log backend so Ankra AI can read logs directly - powering log-aware root-cause analysis and in-product search without leaving the cluster view.

| Backend | Notes |
|---------|-------|
| **Loki** | Grafana Loki HTTP API |
| **Elasticsearch** | Elasticsearch query API |
| **OpenSearch** | OpenSearch query API |

<Steps>
  <Step title="Open the integration">
    Go to the cluster's **Settings → Integrations → Log Source**.
  </Step>
  <Step title="Choose a backend and endpoint">
    Select Loki, Elasticsearch, or OpenSearch and enter the endpoint and any credentials.
  </Step>
  <Step title="Test the connection">
    Ankra validates the endpoint and reports a live connection status.
  </Step>
</Steps>
