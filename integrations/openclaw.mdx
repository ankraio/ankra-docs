## OpenClaw is now Hermes

[OpenClaw](https://openclaws.io/) is the open-source autonomous AI agent runtime
that powers **Hermes** -- Ankra's infrastructure agent. The OpenClaw integration
for Ankra now lives under Hermes, with the same deployment mechanics plus new
capabilities: skills, the Ankra CLI, Ankra AI, stack building, profiles, and
full application infrastructure design with confirmations and a detailed bill of
materials.

### What you can do with OpenClaw on Ankra

- Deploy the OpenClaw runtime to any Kubernetes cluster with a single Ankra stack
- Add the Ankra CLI as an OpenClaw skill to manage clusters from Slack, Discord, or Telegram
- Ask Ankra AI about live cluster state for health checks and troubleshooting
- Build stacks and design application infrastructure -- databases, caches, ingress -- through natural language
- Keep autonomy safe with confirmations, drafted architecture, and a detailed bill of materials

Everything above is documented in the [Hermes integration guide](/integrations/hermes).

For terminal workflows, the Ankra CLI can generate a cluster-specific OpenClaw skill and hand off a conversation to the Ankra portal:

```bash
ankra openclaw skill
ankra openclaw handoff <conversation-id>
```

See the [Ankra CLI reference](/integrations/ankra-cli#openclaw-integration) for flags and examples.
