---
title: Getting Started
description: Essential steps to set up and manage infrastructure efficiently with Ankra
---

We aim to make navigating the Ankra Platform a self-explanatory experience. If you're new, this quick start guide will walk you through the essential steps to set up and manage infrastructure efficiently.

<Tabs>
  <Tab title="Import Existing Cluster">
    <Steps>
      <Step title="Import Cluster">
        For existing Kubernetes environments, importing any cluster allows for immediate visibility and management.
        <ul>
          <li>Navigate to <b>Clusters</b></li>
          <li>Select <b>Import Cluster</b></li>
          <li>Define your initial infrastructure components</li>
          <li>Run the Helm agent command provided on any Kubernetes cluster (including local laptop)</li>
          <li>Once a cluster is imported, maintain or add new infrastructure components through the <b>Add-ons</b> section</li>
          <li>If you disconnect from Ankra, values files are stored in a repository of your choice</li>
        </ul>
      </Step>
      <Step title="Connect to GitHub">
        Enable Values files Ankra creates to be stored in a repository you own.
        <ul>
          <li>Navigate to <b>Integrations</b></li>
          <li>Select <b>Connect GitHub</b> and authorize access</li>
          <li>Choose the repository to use</li>
        </ul>
      </Step>
      <Step title="Deploy Add-ons">
        Extend functionality by deploying essential add-ons.
        <ul>
          <li>Navigate to <b>Add-ons</b></li>
          <li>Browse available options and select desired add-ons to subscribe to</li>
          <li>Once subscribed, deploy the add-on with a single click using default values</li>
          <li>To set custom values, create a profile for your specific use case</li>
        </ul>
      </Step>
      <Step title="Create Profiles">
        Standardize infrastructure by defining reusable profiles for yourself, your organization, or the public.
        <ul>
          <li>Go to <b>Add-ons &gt; Subscribed</b></li>
          <li>Click the Add-on &gt; Profiles &gt; Create</li>
          <li>Set default values for the add-on; templates can enforce or allow configuration by consumers</li>
          <li>Save for future use</li>
        </ul>
      </Step>
      <Step title="Share Infrastructure Profiles">
        Ensure consistency across teams by sharing predefined profiles.
        <ul>
          <li>Open <b>Profiles</b></li>
          <li>Select a profile and choose <b>Share</b></li>
          <li>Define access permissions and distribute as needed</li>
        </ul>
      </Step>
    </Steps>
  </Tab>
  <Tab title="Create New Cluster">
    <Note>
      Ankra provisions vanilla clusters with deeper automation than traditional EKS, providing more control and flexibility.
    </Note>
    <Steps>
      <Step title="Create Cluster">
        For new environments, provisioning a cluster is straightforward.
        <ul>
          <li>Navigate to <b>Clusters</b></li>
          <li>Select <b>Create Cluster</b> (AWS supported today)</li>
          <li>Configure settings such as region, size, and Kubernetes version</li>
          <li>Confirm and provision</li>
        </ul>
        <Note>
          If you disconnect from Ankra, the cluster remains fully functional in your AWS account.
          Any Add-ons installed continue running as defined in the values files, which are stored in a repository of your choice.
        </Note>
      </Step>
      <Step title="Create Credentials">
        Secure access to cloud resources by adding credentials.
        <ul>
          <li>Go to <b>Credentials</b></li>
          <li>Select the cloud provider and input required details</li>
          <li>Save the credentials for use in deployments</li>
        </ul>
      </Step>
