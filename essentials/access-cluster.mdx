<Steps>
  <Step title="Access the Cluster">
    <ul>
      <li>
        <b>Navigate to the <code>Access</code> page</b> in the platform.
      </li>
      <li>
        <b>Download private key:</b> Get the SSH key from the platform.
      </li>
      <li>
        <b>Move Private Key:</b>
        ```bash
        mv ~/Downloads/(unique_key_provided_in_platform.pem) ~/.ssh/
        ```
      </li>
      <li>
        <b>Lock access to private key:</b>
        ```bash
        chmod 0400 ~/.ssh/unique_key_provided_in_platform.pem
        ```
      </li>
      <li>
        <b>SSH to the cluster:</b>
        ```bash
        ssh -i ~/.ssh/command_provided_in_platform_private_key -L 127.0.0.1:6666:api-kubernetes.ankra_command_provided_in_platform:6666 admin@13.13.13.131
        ```
      </li>
    </ul>
  </Step>
  <Step title="Connect to the Cluster with kubectl">
    <ul>
      <li>
        <b>Download kubeconfig:</b> The YAML file contains information for connecting to the cluster and specifies how <code>kubectl</code> interacts with it.
      </li>
      <li>
        <b>Open new terminal</b>
      </li>
      <li>
        <b>If the <code>.kube</code> folder doesn't already exist, create it:</b>
        ```bash
        mkdir -p ~/.kube
        ```
      </li>
      <li>
        <b>Move the kubeconfig file to the <code>.kube</code> folder in your home directory:</b>
        ```bash
        mv ~/Downloads/command_provided_in_platform_kubeconfig ~/.kube/command_provided_in_platform_kubeconfig
        ```
      </li>
      <li>
        <b>Lock access to the kubeconfig file:</b>
        ```bash
        chmod 0600 ~/.kube/command_provided_in_platform_kubeconfig
        ```
      </li>
      <li>
        <b>Interact with the cluster:</b>
        ```bash
        kubectl --kubeconfig ~/.kube/command_provided_in_platform_kubeconfig get pods -A
        ```
      </li>
    </ul>
  </Step>
</Steps>
