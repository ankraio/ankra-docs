## ankra org

Commands to list, switch, create, and manage organisations.

## ankra org assign

Assign a built-in or custom role to a member.

--scope selects where the role applies:
  org (default)      the whole organisation
  cluster:&lt;name&gt;     one cluster
  group:&lt;slug&gt;       every cluster in a cluster group

Examples:
  ankra org assign dev@example.com --role operator --scope group:prod-fleet
  ankra org assign dev@example.com --role platform-on-call --scope cluster:staging-1
  ankra org assign lead@example.com --role admin

```bash
ankra org assign <member-email> [flags]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--role` |   | Role slug or custom role name to assign |
| `--scope` | `org` | Assignment scope: org, cluster:&lt;name&gt;, or group:&lt;slug&gt; |

## ankra org cluster-groups create

Create a cluster group.

Without --selector the group is static: pin clusters afterwards with
'ankra org cluster-groups add-cluster'. With --selector the group is
dynamic: clusters whose labels match every key=value pair are members
automatically (the environment label seeds from the cluster's environment).

Examples:
  ankra org cluster-groups create "Production fleet" --selector environment=production
  ankra org cluster-groups create staging-pool

```bash
ankra org cluster-groups create <name> [flags]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--selector` | `[]` | Label selector pair key=value (repeatable); makes the group dynamic |
| `--slug` |   | Slug for the group (defaults to a slugified name) |

## ankra org invite

Invite a user by email to the current organisation.

Valid roles: owner, admin, operator, member (default), viewer, read-only.
owner/operator alias onto admin/member on the invite until the RBAC
assignments API ships; run 'ankra org roles' for the full list.

Examples:
  ankra org invite user@example.com
  ankra org invite user@example.com --role admin
  ankra org invite user@example.com --role viewer

```bash
ankra org invite <email> [flags]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--role` | `member` | Role for the invited user (owner, admin, operator, member, viewer, read-only) |

## ankra org roles create

Create an organisation custom role.

--permission grants platform permissions (repeatable; 'domain.action'
strings). --kube-access bundles Kubernetes access that is provisioned
automatically on every cluster an assignment of this role covers
(repeatable; '&lt;role&gt;' for whole-cluster or '&lt;role&gt;:&lt;namespace&gt;' for one
namespace, with role one of view, edit, admin, cluster-admin).

Examples:
  ankra org roles create "Platform on-call" \
    --permission clusters.operate --permission stacks.deploy \
    --kube-access edit:app-services --kube-access view

```bash
ankra org roles create <name> [flags]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--kube-access` | `[]` | Kubernetes access '&lt;role&gt;' or '&lt;role&gt;:&lt;namespace&gt;' (repeatable; view\|edit\|admin\|cluster-admin) |
| `--permission` | `[]` | Platform permission 'domain.action' (repeatable) |
| `--slug` |   | Slug for the role (defaults to a slugified name) |

## ankra org variables

Manage organisation-scoped variables that are available to every cluster
in the organisation as template substitutions in stack manifests and addon
values.

  ankra org variables list
  ankra org variables get DB_HOST
  ankra org variables set DB_HOST db.prod.example.com --description "Primary database"
  ankra org variables delete DB_HOST

Variable resolution order at deploy time is stack &gt; cluster &gt; organisation; a
more specific scope overrides a less specific one.

## ankra org variables set

Create or update an organisation variable (upsert). If the variable
does not exist it is created; otherwise its value (and description, when
supplied) is updated.

The value can also be read from stdin by passing "-":

  echo "secret-token" | ankra org variables set API_TOKEN -

```bash
ankra org variables set <name> <value> [flags]
```

**Flags**

| Flag | Default | Description |
|------|---------|-------------|
| `--description` |   | Optional human-readable description |
